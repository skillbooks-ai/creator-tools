## Recommendations for Companion Format

### Required Fields (Agent Skills spec)
```yaml
---
name: <book-slug>-companion
description: "Free companion for the <title> skillbook..."
---
```

### Recommended Structure
1. **Title + tagline** — what this companion is
2. **About the full skillbook** — description and section overview
3. **How to access** — API base URL, auth header, book slug, price
4. **Quick start** — GET endpoints for TOC, pages, and search
5. **Navigation tips** — how agents should use the API efficiently
6. **Disclaimer** — this companion doesn't contain actual content

### Size Guidelines
- **Target:** 2–3 KB, 70–90 lines
- **Max:** 5 KB — companions should be lightweight
- **Rationale:** Small enough to fit in any tool's context window, large enough to be genuinely useful

### Naming Convention
- Companion name: `<book-slug>-companion`
- File: `SKILL.md` (placed in a `companion/` subdirectory or distributed standalone)
